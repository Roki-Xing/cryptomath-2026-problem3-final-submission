# VT/VE Provenance Status

`docs/OFFICIAL_SPEC_INTERPRETATION.md` is the canonical authority for
classifying official, conservative, and unresolved statements. This file
records the repository's frozen-value provenance implementation and does not
expand the official claims defined there.

## Status

**UNRESOLVED official provenance question.**

The frozen baseline currently contains equal numeric strings in its historical
`VT` and `VE` fields. The certified way-2 theorem can explain why these
numerical values are equal, but it does not prove that the submitted `VT`
field was actually generated by way 1.

After PR #9, the repository state is `FULL_EXACT_WAY2_CLOSED`: full exact-way2
recomputed all 4760 unique `(r,u)` columns with both `cpp_int` and
`int128_checked`, and all 138338 frozen endpoints compared as `EXACT_EQUAL`.
This closes the way-2 mathematical and numerical evidence chain only.

After PR #10, Strategy-B Stage-A is `STAGE_A_PASS`: bounded way-1 batch
tooling, query families, shard reduction, compiler gates, and sanitizer gates
passed. Stage-A did not run the full `2^32` plaintext domain, did not run all
138338 way-1 queries, and did not generate a Strategy-B final file.

The current historical equality is therefore a numerical fact, not an
officially cleared provenance claim. The official material made available to
this repository does not establish that a certified way-2 value may replace an
actually executed way-1 `VT`. Strategy B remains the formal low-risk target:
freeze way-2 `VE` first, then compute way-1 `VT` afterwards with a separate
program, input file, schema, and provenance chain.

## Definitions

- **Way 1** enumerates the full `2^32` plaintext domain and produces an exact
  integer numerator over denominator `2^32`.
- **Way 2** propagates correlation masks through the S-box and invertible
  linear layers, aggregating route-shell contributions.
- **Submitted VT field snapshot** means only the historical text currently
  present in `submit.txt`. It is not renamed to `way1_vt`.
- **Frozen way-2 VE** is the original `VE` text from `submit.txt`.
- **Future way-1 VT** remains empty and has status `NOT_EXECUTED` until a
  post-freeze full-domain way-1 pipeline actually runs.

## Current Repository Boundary

The current final generation chain:

- builds candidates and `VE` from way 2;
- does not read `exact_oracle` or `exact_batch_mt` output;
- uses way-1 programs only for the separately stored 18-row spotcheck;
- does not claim that the historical submitted `VT` field came from way 1.

Future Strategy-B work may execute way 1 only after the frozen query and value
snapshot is fixed. It must not use way-1 results to add replacement candidates,
change frozen `VE`, or overwrite the historical `submit.txt`.

## Frozen Evidence

- `experiments/frozen/final_queries.csv`: query coordinates only.
- `experiments/frozen/final_ru.csv`: unique way-2 work units.
- `experiments/frozen/final_values_snapshot.csv`: stable row IDs, historical
  submitted-field text, frozen way-2 `VE`, and empty future way-1 fields.
- `experiments/frozen/BASELINE.json`: submit hash/blob, source commit,
  freeze-tool commit, fixed UTC generation time, counts, and artifact hashes.
- `experiments/frozen/SHA256SUMS.txt`: hashes for all frozen artifacts.
- `artifacts/way2_exact/full/`: full exact-way2 summary, compare,
  authorization, provenance, manifest, SHA, and raw evidence index.
- `artifacts/strategy_b/stage_a/`: Strategy-B Stage-A bounded way-1 batch
  toolchain evidence. It is not full way-1 `VT` provenance.
