## 字节级重建

最终文件由一轮枚举结果和包内已保存的方式二认证来源 CSV 构造：

```bash
python3 -X utf8 experiments/build_submit_from_sources.py --out /tmp/submit_rebuilt.txt
cmp submit.txt /tmp/submit_rebuilt.txt
./score --dedup uv --positive-only submit.txt
```

`experiments/SOURCE_MANIFEST.csv` 列出构造脚本默认读取的全部来源 CSV、参数、行数和 SHA-256。

## 复现层次说明

本作品包支持：

1. 使用已保存的方式二认证来源 CSV，字节级重建最终 `submit.txt`；
2. 重新编译并运行评分、全量审计摘要检查、复杂度汇总和独立抽样脚本；
3. 使用 `exact_oracle` / `exact_batch_mt` 进行 validation-only 精确抽样。

本包不声称包含“从零重新运行全部历史候选搜索”的单命令脚本。最终来源包括 active-1、active-2 edge、active-3 和 E06 r=3 active-2 多类批次；这些已认证来源以 CSV 形式完整保存在包内。方式一程序不参与候选生成、排序、筛选或最终字段生成。

## 工具边界

- `estimator`、`candidate_miner_approx`、`enumerate_r1_positive`：方式二主链。
- `exact_oracle`、`exact_batch_mt`、`reduce_exact_parts`：仅用于独立验证。
- `search_candidates`、`experiments/build_submit_with_certified_r2.py`、`experiments/verify_top_candidates.py`：legacy utility，不属于当前最终生成链；为保持 Makefile 与检查脚本可运行而保留源码。

## 完整性校验

`SHA256SUMS.txt` 覆盖除其自身外的全部包内文件。论文或其他正式文件发生变化后，必须重新生成该文件。

论文 PDF 与 TeX 的 SHA-256 已和提供方清单核对一致。当前 WSL 环境缺少 TeX 指定的物理中文字体，
因此未在本机重新导出 PDF；正式 PDF 已通过页数、标题和最终数字检查。
